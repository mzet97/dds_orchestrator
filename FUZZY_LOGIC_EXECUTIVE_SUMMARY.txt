================================================================================
FUZZY SYSTEM DESIGN
================================================================================

Inputs (4):
  1. urgency (1-10):      Task priority / request urgency
  2. complexity (1-10):   Estimated prompt size / computation need
  3. agent_load (0-100%): Percentage of slots occupied
  4. agent_latency (0-2000ms): Historical average response latency

Outputs (3):
  1. agent_score (0-100): Suitability of candidate agent
  2. qos_profile (0-10): Maps to low_cost, balanced, critical
  3. strategy (0-10): Maps to single, retry, fanout

Rules: 18 Mamdani fuzzy rules
Inference: ~0.3ms per agent evaluation
System Build: ~50ms (one-time)

Example Decision:
  Input: urgent=8, complex=7, load=45%, latency=150ms
  → agent_score=85.2, qos=critical, strategy=retry
  → Select agent-1, use retry strategy if needed, assign critical resources
